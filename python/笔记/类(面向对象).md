# 1.类的方法、定义和使用

```python
color = '黄色'
weight = 10
name = '旺财'

def bark():
	print('汪汪')
	
def sleep():
	print('睡觉')
	
def eat():
	print('吃饭')
```

上述代码区中是面向过程，我们要把这个封装成为一个类

在 Python 中定义类的基本形式为：
```python
class 类名(父类名):
    pass
```
- **class** 是定义类的关键字；
    
- **类名** 处使用符合规范的名称；
    
- **父类名** 是指该类继承的父类名称，如果没有可以连同括号都不写；
    
- **pass** 表示空语句，什么也不做，常用来预留语句位置或临时未写等待以后完成。
    

注意：类在定义后必须先实例化才能使用。

# 2.对象的属性
对象的属性就是`self.xxx` 对象的属性属于对象，必须要实例化才能拿到这个值；类的属性则是不在函数里的属性，类的属性是公有的，不需要实例化就能拿到这个属性的值，且所有的类共享这个值

# 3.类的属性

为了区分类中的变量与全局变量，将类中的变量称为**属性**。

Python 中的类的属性有两种：

- **实例属性**：即同一个类的不同实例的属性，他们的值是不会互相影响的，定义时使用 **实例名.属性名**。（实例属性定义需要带self. ，即def下的定义）
    
- **类属性**：是所有同一个类的实例共有的，直接在类中独立定义，引用时要使用 **类名.属性名** 的形式。（类属性不需要self. ）

```python
class Dog:
	legs=4
	eyes=2
	
# 实例化 （类属性）
my_dog=Dog()
your_dog=Dog()

# 这时候my_dog.leg和my_dog.eyes …… 都有了值

#添加实例属性
my_dog.name='旺财'
my_dog.color='黄色'
my_dog.weight= 10 #这时候my_dog这个实例里面就有了your_dog里面没有的东西

# 修改类的属性
Dog.legs = 2 #这时候所有的类里面的legs都会变成2，包括已经实例化的！！

# 修改实例属性
my_dog.legs = 1 #只修改my_dog.leg的值，your_dog.leg不会变
```

# 4.类的方法

为了区分在类中定义的函数和类外定义的全局函数，将类中定义的==函数称为方法。==

类中的方法定义与函数基本相同，区别有：

- 方法中的第一个参数必须是 self，而且不能省略；
    
- 方法的调用需要实例化类，并以 `实例名.方法名` 形式调用；
    
- 整体进行一个单位的缩进，表示其属于类中的内容。
    

注意：定义方法时，也可以像定义函数一样声明各种形式的参数；方法调用时，不用提供 self 参数。使用方法时，必须要实例化，然后用实例去调用

```python
class Dog:  
    def sleep(self):  
        print("Sleeping")  
  
    def eat(self, food):  
        print(f"Eat {food}")  
  
# 实例化  
my_dog = Dog()  
my_dog.sleep()  
my_dog.eat("apple") # 这就是一个比较完整的定义方法
```


在 Python 中的类定义中，可以定义一个特殊的构造方法，

即：**__init__**( ) 方法

用于类实例化时，初始化相关数据，如果在这个方法中有相关参数，则实例化时就必须提供。且一旦实例化，就立刻执行这个函数

这个方法适用于在类实例化的时候传入相关数据

```python
class Dog:  
    def __init__(self, name):  
        self.name = name  
  
    def sleep(self):  
        print("Sleeping")  
  
    def eat(self, food):  
        print(f"Eat {food}")  
  
    def speak(self):  
        print(f"name: {self.name}")  
  
# 实例化  
my_dog = Dog("Bob")  # 这里必须写一个参数，因为init方法里面有一个name  位置传参  
my_dog.speak()

your_dog = Dog(name="Jack")  #关键字传参
your_dog.speak()
```

注意:
```python
class Dog:
    __legs = 4  # 这是在类中定义的
    
    def print_legs(self):
        # 作用域查找顺序：
        print(__legs)        # ❌ 只在当前函数内部查找（局部作用域）

```

# 5. 面向对象的特征

面向对象编程的三大特性：继承、封装、多态。

- **多态**：  
    多态是指同一个方法调用由于对象不同会产生不同的行为。
    
- **封装**：  
    隐藏对象的属性和实现细节，只对外提供必要的方法。  
    通过私有属性、私有方法的方式，实现封装。Python 追求简洁的语法。
    
- **继承**：  
    继承可以让子类具有父类的特性，提高了代码的重用性。


```python
# 私有属性 默认情况下python中的属性是公有的
class Dog:  
    __legs=4 #私有属性，无法从外部访问
    leg=4
    
    def print_Dog_legs(self):  # 通过方法来调用私有属性
	    print(f'狗的腿数有{self.__legs}')
    
print(Dog.legs) #这时候会报错，说Dog中没有legs这个属性，因为legs是私有属性
print(Dog.leg)  #会输出4
my_dog=Dog()
my_dog.print_Dog_legs() #在调用公用的方法来输出私有属性
```

私有方法： 其实就是一个套娃的过程
```python
# 私有属性 默认情况下python中的属性是公有的
class Dog:  
    __legs=4 #私有属性，无法从外部访问
    leg=4
    
    def __print_Dog_legs(self):  
	    print(f'狗的腿数有{self.__legs}')
	    
	def print_Dog_legs_outside(self):
		self.__print_Dog_legs()  #需要加上双下划线
    
print(Dog.legs) #这时候会报错，说Dog中没有legs这个属性，因为legs是私有属性
print(Dog.leg)  #会输出4
my_dog=Dog()
my_dog.print_Dog_legs_outside() #在调用公用的方法来输出私有属性
```

==注意：python中并没有绝对的私有属性，如果你写一个下划线，则是约定俗称的私有属性,但是如果偏要访问，如`my_dog._leg`依旧可以访问，如果两个下划线，则会对该属性进行名称修饰，`my_dog.__leg`会报错，因为名字被修饰了，`my_dog._Dog__leg`则可以正常访问(`实例化名._类名__私有属性`)。==

## 继承

```python
class Animal:  
    name = '动物'  
    def sleep(self):  
        print('sleeping')  
          
class Dog(Animal):  
    pass  
  
my_dog = Dog()  
print(my_dog.name)  
my_dog.sleep()
```

这就是一个最简单的继承，Dog类（子类）继承了Animal（父类）

重写，即如果子类中有父类的属性，则子类调用会覆盖父类

```python
class Animal:  
    name = '动物'  
    def sleep(self):  
        print('sleeping')  
          
class Dog(Animal):  
    name = '狗' 
    
    def sleep(self):  
        print(' need sleeping')  
  
my_dog = Dog()  
print(my_dog.name)  
my_dog.sleep()
```

对于init的构造方法依旧实用，子类的调用，如果没有init函数，会调用父类的构造函数

```python
class Animal:  
    def __init__(self, name):  
        self.name = name  
          
class Dog(Animal):  
    def __init__(self, name, age):  
        self.name = name  
        self.age = age  
          
my_dog = Dog('Bob', 2)  
print(my_dog.name, my_dog.age)
```

最后会输出子类的init调用，因为子类的init覆盖了父类的init

```python
class Dog(Animal):  
    pass  
  
my_dog = Dog('Bob')  
print(my_dog.name)
```

依旧会输出 Bob

super( ) 函数的使用

```python
class Animal:  
    def __init__(self, name):  
        self.name = name  
  
class Dog(Animal):  
    def __init__(self, name, age):  
        super().__init__(name)  
        self.age = age  
  
my_dog = Dog('Bob', 2)  
print(my_dog.name, my_dog.age)
```

依旧可以运行成功，这是因为super函数继承调用了父类的init构造中的name，使得子类并不会完全覆盖父类，如果用`super().__init__()` 则会调用父类初始化，父类的init方法会继承到子类的init中，如果父类有些东西不想被子类及继承，比如name，只需要设置为私有属性就可以

```python
class Animal:  
    def __init__(self, name):  
        self.__name = name  
  
class Dog(Animal):  
    def __init__(self, name, age):  
        super().__init__(name)
        self.name =name  
        self.age = age  
  
my_dog = Dog('Bob', 2)  
print(my_dog.name, my_dog.age)
```

这个name的来源来自于子类

# 6. classmethod 装饰器

对于类里面的每一个函数，都是对象函数，并不是类的函数，其典型特征就是函数需要实例化才可以使用，我们可以通过装饰器来使得这个函数变为类的函数，从而不实例化也能使用这个函数

```python
class Dog():  
    @classmethod  
    def piss(self):  
        print('piss')  
  
    def cheese(self):  
        print('cheese')  
  
Dog.piss() # 这个调用是对的，但并没有实例化，通过装饰器装饰就可以，且cheese不会受到影响
```

## 装饰器的本质

装饰器的本质就是一个函数，把被装饰的函数当成参数
```python
import time

def timer(func):
	def inner(*args, **kwargs):
		start = time.time()
		func(*args, **kwargs)
		print(time.time() - start)
	return inner
	
@timer
def Test():
	for i in range(10000):
		print(123)
		
Test()
#上面的函数就是一个装饰器
```

# 7. 自定义模块和__main__

```python
# testlib.py
def add(a, b):
	print(a+b)
	
add(1, 3)
```

```python
# uselib.py
import testlib
testlib.add(2, 3)
# or
from testlib import add
add(2, 3) # 这时候就不认识testlib了，只认识add
```
但是在uselib中的这两种运行方式在输出的时候都会输出4 5
因为不论是import还是from import ，原理都是去运行这个模块里的所有代码，只需要删除add(1, 3)就可以了

如果你就是想保留add（1，3），方便调试
这时候更改testlib
```python
def add(a, b):
	print(a+b)

if __name__ == '__main__':
	add(1, 3)
```

解释：
	如果你在uselib去调用testlib去使用，这时候的`__name__ ==__testlib__ `，即文件名，如果是在testlib里面去运行，即自己运行自己，`__name__ == __main__`
