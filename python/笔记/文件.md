
目的共有五种
- 文本读（r）  文本写（w）  二进制读（rb）  二进制写（wb）  追加写（a）
- 注意，二进制读通常被使用与图片，.mp3，等文件

```python
# 1.打开文件
f = open('./class.py', 'r') #open函数会返回一个文件的对象给我们，这里记为f (./是指的该路径)

# 2.读写文件
print(f.read()) #会读取全部的内容
```

如果这个时候报错了，并且含有gbk报错，大部分是文件里有中文，想要解决也简单
```python
# 1.打开文件
f = open('./class.py', 'r', encoding='utf8') #open函数会返回一个文件的对象给我们，这里记为f (./是指的该路径)

# 2.读写文件
print(f.read())
```

如果你想一行一行的读，就改为`f.readline()`，也可以用循环来一行一行读全文
```python
while True:
	data = f.readline() #读取出来的数据是str
	if len(data)==0:
		break
	print(data)
```
也可以把每一行的数据都放到一个列表里
```python
data = f.readlines()
print(data)
```

在读写完后需要关闭文件 `f.close()` ，读是感觉不到差别的，写的时候有无关闭文件很重要

```python
f = open('./text.txt', 'w', encoding='utf8') #不存在就会新建;存在再用w去写入，就会覆盖原文件内容

f.write('Mythalis')
f.write('MxTwinkle_w')  #不会换行，会累加在Mythalis后面，需要加换行符\n
f.writelines(['\n请大家多多支持\n', 'Mythalis'])  #写入列表里面的数据

f.close()
```

如果把open的w改为a，就不会覆盖数据了，只会追加

如果没有 `f.close()` 中间再加上死循环，数据在结束程序之前是不会写入的
如有没有`f.close()`  中间没有死循环，数据会写入，程序结束就是文件关闭
如果如下
```python
f = open('./text.txt', 'w', encoding='utf8') #不存在就会新建;存在再用w去写入，就会覆盖原文件内容

f.write('Mythalis')
f.write('MxTwinkle_w')  #不会换行，会累加在Mythalis后面，需要加换行符\n
f.writelines(['\n请大家多多支持\n', 'Mythalis'])  #写入列表里面的数据
f.close()
While True:
	pass
```

程序会卡住，但是文件依旧会写入   


如果你总是忘记去写这个close，也有一个方法，就是使用with

```python
with open('./text.txt', 'w', encoding='utf8') as f: #不存在就会新建;存在再用w去写入，就会覆盖原文件内容
	f.write('Mythalis')
	f.write('MxTwinkle_w')  #不会换行，会累加在Mythalis后面，需要加换行符\n
	f.writelines(['\n请大家多多支持\n', 'Mythalis'])  #写入列表里面的数据
While True:
	pass
```
这时候就不需要写`f.close()`
