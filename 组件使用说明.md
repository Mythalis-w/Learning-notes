# 工业缺陷检测管理系统组件使用说明（前后端）

## 1. 文档目的

本文档用于说明系统前后端主要组件/模块的职责、调用关系与使用方式，便于开发、测试、运维和交付验收阶段统一理解与沟通。

## 2. 技术栈与架构声明

- 前端：`React`（Create React App）+ `Ant Design` + `react-router-dom`
- 后端：`Go`（Golang）标准库 `net/http`
- 数据库：`SQLite`
- 接口通信：前端通过 REST API 直接调用后端，统一前缀为 `/api/`
- 架构原则：后端不依赖 Gin、Echo、Beego 等第三方 Web 框架，核心 Web 服务能力由 Go 标准库与自研业务代码实现

## 3. 开源合规性说明

- 项目依赖来源于公开开源生态（如 npm、Go Modules）。
- 各依赖组件在使用过程中遵循其许可证要求（如声明保留、用途合规等）。
- 建议在交付与审计阶段同步提供依赖清单与许可证信息，确保可追溯和可核验。

## 4. 前端组件说明（React）

### 4.1 通用组件（`frontend/src/components`）

- `DefectList.js`：缺陷列表展示组件，负责列表渲染、筛选和操作入口承载。
- `DefectForm.js`：缺陷创建/编辑表单组件，负责字段录入与提交。
- `ProtectedRoute.js`：路由权限组件，控制未登录或无权限用户访问受限页面。
- `MasterDataManagePage.js`：通用主数据管理组件，可复用于产线、设备、缺陷类型等 CRUD 页面。

### 4.2 页面组件（`frontend/src/pages`）

- `DashboardPage.js`：系统首页与统计看板。
- `LoginPage.js`：登录页面。
- `DefectListPage.js`：缺陷列表页。
- `DefectDetailPage.js`：缺陷详情页。
- `DefectCreatePage.js`：缺陷创建页。
- `ProductionLineManagePage.js`：产线管理页。
- `EquipmentManagePage.js`：设备管理页。
- `DefectTypeManagePage.js`：缺陷类型管理页。
- `UserManagementPage.js`：用户管理页。
- `OperationLogsPage.js`：操作日志页。

### 4.3 前端基础模块

- `App.js`：应用路由与页面挂载入口。
- `index.js`：前端应用启动入口。
- `api.js`：HTTP 请求封装（统一请求地址、鉴权头与异常处理）。

## 5. 后端模块说明（Go）

### 5.1 核心业务模块

- `backend/main.go`：负责认证、用户、缺陷流程、日志、导出等核心接口与主流程控制。

### 5.2 主数据模块

- `backend/masterdata.go`：负责产线、设备、缺陷类型等主数据接口与管理逻辑。

### 5.3 测试模块

- `backend/main_test.go`：使用 `httptest` 进行接口行为与关键流程测试，保障核心功能稳定性。

## 6. 数据库说明（SQLite）

- 数据库类型：嵌入式 `SQLite`
- 主要作用：存储缺陷单、用户、会话、主数据、操作日志、流程日志等业务数据
- 特点：部署轻量、便于本地运行与维护

## 7. 前后端调用关系

1. 前端页面/组件通过 `api.js` 发起 API 请求。
2. 后端基于 Go 标准库处理 `/api/*` 接口请求。
3. 后端执行业务逻辑并读写 SQLite。
4. 后端返回 JSON，前端完成页面渲染与交互反馈。

## 8. 使用与维护建议

- 新增主数据类页面优先复用 `MasterDataManagePage.js`，减少重复开发。
- 新增接口保持 `/api/` 前缀统一，便于鉴权、日志和权限管理。
- 新增第三方依赖时同步评估许可证风险，并更新依赖与合规说明。
- 对外技术说明统一采用本文档口径，明确“无第三方 Web 框架依赖”。
