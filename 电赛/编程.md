
# 1.点灯 /FPIOA和GPIO

**红灯**，**绿灯**，**蓝灯**分别接到了**GPIO62**，**GPIO20**和**GPIO63**上面
板载RGB灯是共阳结构，设置引脚为高电平时关闭灯，低电平时点亮灯

pin的上拉输出会让该引脚处于默认的高电平状态
而下拉输出会处于默认的低电平状态

```python
from machine import FPIOA  #用于初始化引脚，配置引脚的功能需要去官网查找
from machine import Pin  #实例化引脚

# 创建FPIOA对象，用于初始化引脚功能配置
fpioa = FPIOA()

fpioa.set_function(62,FPIOA.GPIO62)
fpioa.set_function(20,FPIOA.GPIO20)
fpioa.set_function(63,FPIOA.GPIO63)
#将芯片的第62个物理引脚（或称为引脚复用控制器上的第62号功能复用通道）的功能，设置为“作为普通的GPIO（通用输入/输出）使用”。

# 实例化Pin62, Pin20, Pin63为输出，分别控制红、绿、蓝灯
LED_R = Pin(62, Pin.OUT, pull=Pin.PULL_NONE, drive=7)
LED_G = Pin(20, Pin.OUT, pull=Pin.PULL_NONE, drive=7)
LED_B = Pin(63, Pin.OUT, pull=Pin.PULL_NONE, drive=7)
'''
Pin.OUT是由代码控制外部的设备，如控制RGB灯
Pin.IN是接受外部的信号

drive默认为7

pull=Pin.PULL_NONE 这个是默认值
pull=Pin.PULL_UP  这个是上拉，默认引脚为高电平
pull=Pin.PULL_DOWN  这个是下拉，默认引脚为低电平

如果使用下拉，则LED_R.value() == 0  (LED.R是每一个实例化的名字)
'''

button = Pin(53, Pin.IN, pull=Pin.PULL_DOWN) # 使用下拉电阻 即默认value是0
# 这个按钮是改变53这个引脚的电平，具体控制灯需要用下面的if来判断，判断条件是53的pin的value

# 初始化时先关闭所有LED灯
LED_R.high()  # 关闭红灯
LED_G.high()  # 关闭绿灯
LED_B.high()  # 关闭蓝灯
'''
对于每一个实例化的对象，name.high() 是使这个引脚是高电平，low是低  高电平关灯，低电平开灯
依据设备不同应该也不太一样
'''

LED = LED_G  # 设定一个控制的LED ，不设定也行，写LED_G也不会错

while True:
    if button.value() == 0:
        LED.high()
    else:
        LED.low()
'''
这个循环判断，通过button.value()的value来控制灯光pin的高电平还是低电平
'''
```

## 使用FPIOA功能

1. 构造函数`fpioa = FPIOA()`
2. 设置引脚功能 `fpioa.set_function(pin, func, ie=-1, oe=-1, pu=-1, pd=-1, st=-1, sl=-1, ds=-1)`
	设置引脚功能是为了在引脚复用的时候区分用哪一个，每一个排针有不同的功能
**参数**：

- `pin`: 要配置的引脚编号，范围：[0, 63] ==注意，pin指的是芯片引脚的编号，不是排针的编号==
- `func`: 要分配给引脚的功能代码
    - 普通GPIO：`FPIOA.GPIO0`，`FPIOA.GPIO1`，`FPIOA.GPIO2`等，范围【0，63】
    - 串口：`FPIOA.UART0_TXD`，`FPIOA.UART0_RXD`，`FPIOA.UART1_RXD`等。
    - IIC：`FPIOA.IIC0_SCL`，`FPIOA.IIC0_SDA`，`FPIOA.IIC1_SCL`，`FPIOA.IIC1_SDA`等。
    - PWM： `FPIOA.PWM0`， `FPIOA.PWM1`， `FPIOA.PWM2`等。
    - 以及其他功能，更多详情请参考上方**3.3章**的 **[各引脚可复用功能](https://wiki.lckfb.com/zh-hans/lushan-pi-k230/basic/gpio-fpioa.html#_3-3-%E5%90%84%E5%BC%95%E8%84%9A%E5%8F%AF%E5%A4%8D%E7%94%A8%E5%8A%9F%E8%83%BD)**。

# 2.PWM

先用`FPIOA`的`set_function`方法把对应GPIO(引脚)配置为PWM模式。如下所示：
PWM 类位于 `machine` 模块中。

```python
from machine import FPIOA

# 创建FPIOA对象，用于初始化引脚功能配置
fpioa = FPIOA()

# 将排针处有PWM功能的引脚配置为PWM功能
fpioa.set_function(47, FPIOA.PWM3)
fpioa.set_function(61, FPIOA.PWM1)
fpioa.set_function(46, FPIOA.PWM2)
fpioa.set_function(52, FPIOA.PWM4)
fpioa.set_function(42, FPIOA.PWM0)

# 将蜂鸣器驱动脚GPIO43配置为PWM功能
fpioa.set_function(43, FPIOA.PWM1)

# 将屏幕背光驱动驱动脚GPIO25配置为PWM功能
fpioa.set_function(25, FPIOA.PWM5)
```

简单鸣叫一声

```python
import time
from machine import PWM, FPIOA

# 配置蜂鸣器IO口功能
beep_io = FPIOA()
beep_io.set_function(43, FPIOA.PWM1)

# 初始化蜂鸣器PWM通道，
beep_pwm = PWM(1)  # 默认频率4kHz,占空比50%

# 调整通道0频率为4000Hz
beep_pwm.freq(4000)

# 调整通道0的占空比为 50% (32768 / 65535)
beep_pwm.duty_u16(32768)
# 使能PWM通道输出
# 延时50ms
time.sleep_ms(50)
# 关闭PWM输出 防止蜂鸣器吵闹
beep_pwm.duty_u16(0)
# 叫完了就释放PWM
beep_pwm.deinit()
```

PWM参数是用来设置通道的,第一个参数取值为0-5全闭，并且取1通道的原因是在配置IO口功能的时候PWM1是作为43号引脚（蜂鸣器的通道的） 注意：驱动蜂鸣器的PWM通道是PWM1，而排针处的26号引脚（GPIO 61）的通道也是PWM1，因此它们无法同时独立使用PWM功能。

- `freq`: PWM 通道输出频率
- `duty`: PWM 通道输出占空比，表示高电平在整个周期中的百分比，取值范围为 \[0, 100]
- `duty_ns`: PWM 通道输出高电平的时间，单位为 `ns`
- `duty_u16`: PWM通道输出高电平的时间，取值范围为 \[0,65535]  做比计算（这个是音量大小）

PWM.deinit() 是用来释放PWM的

frep来设置频率，周期为频率的倒数，再通过duty_u16来设置音量（65535是占空比的分母，定义分子就能计算响多大）

注意，开始叫是从设置完 `beep_pwm.duty_u16(32768)`  开始的，这个是设置音量的

==Pin 和 PWM 作为两种模式，各有不同的用处==

# TIMER

总代码
```python
from machine import Timer
import time

# 创建一个软件定时器实例，-1 表示使用软件定时器
tim = Timer(-1)

# 配置定时器，单次模式（ONE_SHOT），定时100毫秒，回调函数打印数字1
tim.init(period=100, mode=Timer.ONE_SHOT, callback=lambda t: print("Timer Callback: 1"))

# 再次配置定时器，这次为周期模式（PERIODIC），每隔1000毫秒（1秒）打印数字2
tim.init(period=1000, mode=Timer.PERIODIC, callback=lambda t: print("Timer Callback: 2"))

# 延时5.1秒，让定时有时间执行
time.sleep(5.1)

# 当不再需要定时器时，可以停用它
tim.deinit()
```

- 构造函数
`timer = Timer(index, mode=Timer.PERIODIC, freq=-1, period=-1, callback=None, arg=None)`

**参数**

- `index`: 定时器模块编号，取值范围为 `[-1, 5]`。
    - `-1` 表示软件定时器。
    - `0~5` 为硬件定时器编号，【但当前不可用】。
    
- `mode`: 定时器运行模式，可选 `Timer.ONE_SHOT` 或 `Timer.PERIODIC`。
    - `ONE_SHOT`：定时器在触发一次回调后自动停止。
    - `PERIODIC`：定时器在触发回调后会自动重置周期，持续周期性触发。
    
- `freq`: 定时器频率（Hz），可以为浮点数。如果设置了 `freq`，则定时器的时间间隔由频率决定，`freq` 优先级高于 `period`。例如，`freq=1` 相当于 `period=1000ms`。

- `period`: 定时器周期，单位为毫秒（ms），如果未设置 `freq`，则以 `period` 为准。

- `callback`: 超时回调==函数==。当定时器计数完成后自动调用该函数，函数至少应包含一个参数（接收定时器自身对象或关联参数）。
  注意一定是一个函数，没有函数就自己定义

- `arg`: 超时回调函数的参数（可选）。如果提供了 `arg`，在回调函数中将可以通过传入的参数获取额外信息。

**注意：** **硬件定时器 [0-5] 暂不可用。**
****
- `init` 方法
`Timer.init(mode=Timer.PERIODIC, freq=-1, period=-1, callback=None, arg=None)`
这个Timer.init前面的Timer是构造函数中等号前面的name

用于初始化或重新配置定时器的参数。参数与构造函数相同。

==注意，这个计时器是异步进行的，会在后台计时，代码不会运行到这里就停止==
**参数**

- `mode`: 定时器运行模式，可以是单次或周期模式（可选参数）。
- `freq`: 定时器运行频率，支持浮点数，单位为赫兹（Hz），此参数优先级高于 `period`（可选参数）。
- `period`: 定时器运行周期，单位为毫秒（ms）（可选参数）。
- `callback`: 超时回调函数，必须设置并应带有一个参数。
- `arg`: 超时回调函数的参数（可选参数）。

**返回值**

无
