# 主体

这个算法广泛应用于各类的进制转换的问题中。实际上，不论是什么进制，转化成十进制无非就是一个多项式

$$
P(x) = d_nx^n + d_{n-1}x^{n-1} + \cdots + d_1x + d_0
$$

无论什么多项式，都可以这样来表示，如下，“631” 这是一个八进制，我们需要转化为十进制，我们先让电脑算一下就会得出答案，即409。写成多项式即 $409 = 6\cdot8^2 + 3\cdot8^1 + 1\cdot8^0$

而实际上，对于任何的一个$P(x)$ ，我们都可以通过转化写成

$$
P(x) = d_0 + x(d_1 + x(d_2 + \cdots + x(d_n)))
$$

为了更容易发现规律，我们可以改变下形式

$$
P(x) = d_0 + x(d_1 + x(d_2 + \cdots + x(d_n+(x\cdot0))))
$$

这是一个迭代的形式，其中 $x$ 对应的是一个x进制，而 $d_n$ 则对应的是第n个数字（第n个数字即最高位数字），也就是说，通过这个算法，我们可以在代码里通过迭代来实现快速的求出一个结果的方法，`result = result * 8 + digit;`(注意：result初始为0) ，如下

```C
#include <stdio.h>
int reverseNumber(int n) {
    if (n == 0) return 0;
    int sign = 1;  //sign是用来判断正负的
    if (n < 0) {
        sign = -1;
        n = -n;
    }
    int reversed = 0;
    while (n != 0) {
        reversed = reversed * 10 + (n % 10);
        n /= 10;
    }
    return reversed * sign;
}

int main(){  // 求 631  八进制转化为十进制 
int translation = 8;
int result = 0;
int num_simple,num;
scanf("%d",&num);
num = reverseNumber(num);
while(num != 0){
	num_simple = num % 10;
	num = num/10;
	result = result * translation + num_simple;   //！！！
	}
printf("%d",result);
return 0;
}
```

看起来很不错，它实现了八进制的求解，但是，这个代码仍然有问题，它无法处理末尾'0'的情况，如“100” =64 ，但是经过代码运行的答案却是 1 ，因为经过反转001变为了1，导致的处理错误，因此，对于进制转化的问题，使用霍纳算法最好不要用反转，使用字符串是最好的

```C
#include <stdio.h>
#include <string.h>

int main() {
    char octal_str[100];
    printf("输入八进制数: ");
    scanf("%s", octal_str);
    
    int result = 0;
    for (int i = 0; i < strlen(octal_str); i++) {
        int digit = octal_str[i] - '0';  // octal_str[i] 本身是一个字符，需要变为数字
        result = result * 8 + digit;
    }
    printf("十进制: %d\n", result);
    return 0;
}
```
