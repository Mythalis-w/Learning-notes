
# 1.类和对象

在C++中，允许使用class和struct两种方法去定义类，其中，class默认所有的类都是私有的，struct默认所有的类都是共有的，除此以外，两者完全等价。

```C++
class Student {
// 默认是 private 区域
    std::string private_name;  // 这个是私有的，类外不能访问

public:  // 从这里开始，后面的成员都是公有的
    std::string name;  // 公有的
    int age;           // 公有的

    void display() {   // 公有的成员函数
        // 在类的内部，可以访问私有成员
        private_name = "内部可访问"; 
    }

private: // 又可以切换回私有区域
    double secret_score; // 又是私有的了
};
```
类访问其中的共有属性或者共有的成员时，使用`class_name.name`

类类型也有指针变量
```C++
student stu;
student *p = &stu;
(*p).name = "Liping"
student student[3];
p = student+2;
```
指针指向动态分配的内存也是可以的
```C++
student *p = new student;
p -> name = "Wangping";
delete p;
p= new student[3];
```

在类中，成员函数可以像成员那样被调用，如`stu.print()`


## 构造函数

这一点和python中的__init__很像，每一个类在创建的时候都会自己生成并调用一个构造函数，我们上面的类并没有实现将参数传入程序的作用，构造函数就可以实现传入参数的作用
注意，构造函数应该放在public中

例子如下
```C++
#include<iostream>  
#include<string>  
using namespace std;  
  
class student{  
private:  
    string name;  
    int id;  
    double score;  
public:  
    student(string s,int i ,double n){  
        name = s; id = i; score =n;  
    }  
    void print(){  
        cout << name <<" "<< id<<" " << score<<endl;  
    }  
};  
  
int main(){  
    string name;  
    int id;  
    double score;  
  
    cout << "name " << "id " << "score";  
    cin >> name >> id >> score;  
  
    student stu(name , id ,score);  
    stu.print();  
    return 0;  
}
```

==如果你想写构造函数，但是只是想写出来，可以`name(){;}`，和大部分类型一样，不想写东西就写一个；== 

在对象建立数组的时候，也要调用构造函数，数组中有几个元素就要调用几次构造函数
```C++
class Point{
	int x,y;;
public:
	Point(int a, int b);
	void print();
};
Point::Point(int a=0, int b=0){
	x=a;y=b;
}

int main(){
	Point p1;
	Point p2[3]={Point(1,1),Point(2,2),Point(3,3)} // 调用三次构造函数
}
```
除了构造函数可以传入参数，类还可以通过初始化列表来赋值,但是这种初始化顺序和声明顺序必须一样，也叫做：初始化列表

```C++
class Example {
    std::string s;
    int i;
public:
    // 低效方式：s先被默认构造为空字符串，然后被赋值为"Hello"
    Example() {
        s = "Hello"; 
        i = 10;
    }

    // 高效方式：s直接通过拷贝构造函数初始化为"Hello"
    Example() : s("Hello"), i(10) {
    }
};
```
## 构造函数的重载

构造函数允许写多个，只要函数标签不同就可以
```C++
class Student {
private:
    string name;
    int age;
    double score;
    
public:
    // 1. 无参构造函数
    Student() {
        name = "Unknown";
        age = 0;
        score = 0.0;
        cout << "无参构造" << endl;
    }
    
    // 2. 单参数构造函数
    Student(string n) {
        name = n;
        age = 0;
        score = 0.0;
        cout << "单参数构造" << endl;
    }
    
    // 3. 双参数构造函数
    Student(string n, int a) {
        name = n;
        age = a;
        score = 0.0;
        cout << "双参数构造" << endl;
    }
    
    // 4. 三参数构造函数
    Student(string n, int a, double s) {
        name = n;
        age = a;
        score = s;
        cout << "三参数构造" << endl;
    }
    
    void display() {
        cout << name << ", " << age << ", " << score << endl;
    }
};

int main() {
    Student s1;                    // 调用无参构造
    Student s2("张三");             // 调用单参数构造
    Student s3("李四", 20);         // 调用双参数构造
    Student s4("王五", 22, 95.5);   // 调用三参数构造
    
    return 0;
}
```

# 2.内联函数

内联函数是C++中的一个特性，用关键字 `inline` 修饰，目的是**减少函数调用的开销**。

- 普通函数调用：需要压栈、跳转、返回等操作
- 内联函数：在==**编译**时**将函数体直接插入**到调用处==，像宏一样展开，但保留了函数的语法特性

```C++
// 普通函数
int add(int a, int b) {
    return a + b;
}
int result = add(3, 5);  // 调用函数，有开销

// 内联函数
inline int add(int a, int b) {
    return a + b;
}
int result = add(3, 5);  // 编译时直接替换为 3 + 5
```

可以消除一部分函数调用的开销
==注意：类内定义的函数通常是默认内联的 ; 而复杂的函数哪怕写了inline也可能不会内联，这个看编译器了,inline修饰只是给编译器建议是否内联，而不是强制。==

# 3. 析构函数

析构函数是C++中一个特殊的成员函数，在对象**被销毁时自动调用**，用于执行清理工作。

- 特性：
- **无参数**，无返回值
    
- **不能重载**（每个类只有一个析构函数）
    
- 对象销毁时**自动调用**
    
- 主要用途：**释放资源**（堆内存、文件句柄、网络连接等）

# 4. 友元函数

**友元函数**（Friend Function）是 C++ 中定义在类的外部，但有权访问类的所有私有（private）成员和保护（protected）成员的普通函数。

其特点就是在类内被声明，但是定义要写在外面。类的内部，使用关键字 `friend` 加上函数的原型进行声明，可以放在类的私有和公有部分，都是可以的。在函数外部定义

```C++
#include<iostream>
using namespace std;

class student{
    int id;
    double grade;
public:
    void setData(int i, double s){
        id = i;
        grade = s;
    }
    friend double average(student *p, int count);
};

double average(student *p, int count){
    if(count == 0) return 0;
    double sum = 0;
    for(int i = 0; i < count; i++){
        sum += (*(p+i)).grade;
    }
    return sum / count;
}

int main() {
    student stu[101];
    int id;
    double score;
    int count = 0;
    
    while (true) {
        cin >> id;
        if (id == 0) break;
        
        cin >> score;
        stu[count].setData(id, score);
        count++;
    }
    
    double avg = average(stu, count);
    cout << avg << endl;
    
    return 0;
}
```

# 5. 类的继承（派生类）

我们首先讲解protected访问修饰符：`protected` 是C++类中的**访问控制修饰符**，它的访问权限介于 `public` 和 `private` 之间。

### 访问权限对比
| 访问修饰符         | 本类内部  | 派生类内部  | 外部（其他函数/对象） |
| ------------- | ----- | ------ | ----------- |
| **public**    | ✅ 可访问 | ✅ 可访问  | ✅ 可访问       |
| **protected** | ✅ 可访问 | ✅ 可访问  | ❌ 不可访问      |
| **private**   | ✅ 可访问 | ❌ 不可访问 | ❌ 不可访问      |

讲完了protected，可以讲派生类了，派生类就是类的继承
语法：
```C++
class 基类 {
    // 基类成员
};

class 派生类 : 访问说明符 基类 {
    // 派生类特有的成员
};
```
访问说明符有三种，即private public protected ，**举个例子，T 继承 就是在基类中public和protected中的所有东西都能被继承，但是所有继承的东西在派生类中都会变成T (即我们必须注意，基类的访问修饰符和派生类的访问说明符并不是一个东西)**
示例：

```C++
#include <iostream>
using namespace std;

class Animal {
public:
    void eat() { cout << "Eating" << endl; }
    
protected:
    void sleep() { cout << "Sleeping" << endl; }
    
private:
    void breathe() { cout << "Breathing" << endl; }  // 永远无法被继承
};

// 默认 private 继承
class Cat : Animal {  // 相当于 private Animal
public:
    void catAction() {
        eat();      // ✅ 可以（变成 private 成员）
        sleep();    // ✅ 可以（变成 private 成员）
        // breathe(); // ❌ 永远不能访问 private 成员
    }
};

// 显式 public 继承（通常想要的方式）
class Dog : public Animal {
public:
    void dogAction() {
        eat();      // ✅ 可以（保持 public）
        sleep();    // ✅ 可以（保持 protected）
    }
};

int main() {
    Cat cat;
    Dog dog;
    
    // Cat 是 private 继承
    // cat.eat();  // ❌ 错误！eat 在 Cat 中是 private
    
    // Dog 是 public 继承
    dog.eat();      // ✅ 可以！eat 保持 public
    
    // dog.sleep();  // ❌ 错误！sleep 是 protected，外部不能访问
    
    return 0;
}
```

# 6. 指针

## this指针

this指针是一个**隐含的指针**，它指向**当前对象本身**。当你在类的成员函数中访问成员变量或其他成员函数时，this指针就在背后起作用。

```C++
class Person {
private:
    string name;
    int age;
    
public:
    Person(string name, int age) {
        // 使用this指针区分参数和成员变量
        this->name = name;  // 左边的name是成员变量，右边的name是参数
        this->age = age;     // this->age表示当前对象的age
    }
    
    void introduce() {
        // 这里实际上隐含了this指针
        cout << "我叫" << this->name << "，今年" << this->age << "岁" << endl;
        // 等价于：cout << "我叫" << name << "，今年" << age << "岁" << endl;
    }
    
    // 返回当前对象的引用
    Person& growOlder() {
        this->age++;
        return *this;  // 返回当前对象本身
    }
};
```

## 对象指针

- 每个对象在初始化后都会在内存中占一定的空间，因此，既可以通过对象名访问一个对象，也可以通过对象地址来访问一个对象；
    `类名 *对象指针名;`
- 使用指向对象的指针来访问对象成员，要用 “`->`” 操作符；
- 每个对象的存储空间只是该对象数据成员所占用的存储空间，函数代码存储在对象空间之外；
- C++ 编译系统只用一段空间来存放共同的函数代码段，在调用各对象的成员函数时，都去调用这个公用的函数代码；

```C++
#include <iostream>
using namespace std;

// 定义Point类
class Point
{
    int x, y; // 数据成员：每个对象独有，占对象内存
public:
    // 构造函数：函数代码是所有对象共用的
    Point(int a = 0, int b = 0) : x(a), y(b) {}
    
    // 成员函数print：函数代码是所有对象共用的
    void print() {
        cout << "(" << x << "," << y << ")" << endl;
    }
};

int main() {
    // 1. 通过对象名访问对象
    Point p1(1, 1);
    cout << "通过对象名p1访问print()：";
    p1.print(); // 用.操作符访问成员

    // 2. 通过对象指针访问对象（对应知识点1、2）
    Point *p2 = &p1; // 定义对象指针，指向p1的地址
    cout << "通过指针p2访问print()：";
    p2->print();    // 用->操作符访问成员（指针专用）
    // 等价写法：(*p2).print(); （先解引用，再用.）

    return 0;
}
```
# 7.静态成员

在 C++ 中，静态成员是属于**整个类**而非类的某个对象的成员，分为**静态成员变量**和**静态成员函数**。可以把它理解为：类的 “共享资源”，所有对象共用这一份数据 / 方法，不随对象创建 / 销毁而变化。

其中，静态成员是写在类里面的，但是静态变量就是写在外面的

下面举一个静态成员变量的例子
```C++
#include <iostream>
using namespace std;

class Student {
public:
    string name;          // 普通成员变量（每个对象独有）
    static int totalNum;  // 静态成员变量（所有学生共享的总人数）
    // 构造函数
    Student(string n) : name(n) {
        totalNum++;  // 每创建一个对象，总人数+1
    }
};

// 静态成员变量类外初始化（语法：类型 类名::变量名 = 初始值） 类外定义
int Student::totalNum = 0;  

int main() {
    Student s1("张三");
    Student s2("李四");
    
    // 两种访问方式：对象访问 / 类名直接访问（推荐）
    cout << "通过对象访问：" << s1.totalNum << endl;  // 输出2
    cout << "通过类名访问：" << Student::totalNum << endl;  // 输出2
    
    // 修改静态变量，所有对象共享
    Student::totalNum = 5;  //对象名.静态变量也是可以访问的
    cout << s2.totalNum << endl;  // 输出5
    return 0;
}
```

## 静态成员函数：
- 属于类而非对象，没有隐含的`this`指针； （因此无法访问普通成员）
- 只能访问**静态成员变量 / 静态成员函数**，不能访问普通成员（因为普通成员依赖具体对象）；
- 可以通过类名直接调用，无需创建对象。

```C++
#include <iostream>
using namespace std;

class Calculator {
public:
    static int sum;  // 静态成员变量：累计求和结果
    // 静态成员函数：计算两数之和并累计
    static int add(int a, int b) {
        sum += (a + b);  // 只能访问静态成员sum
        return a + b;
    }
};

int Calculator::sum = 0;  // 初始化静态变量

int main() {
    // 无需创建对象，直接通过类名调用静态函数
    cout << "1+2=" << Calculator::add(1, 2) << endl;  // 输出3
    cout << "累计和：" << Calculator::sum << endl;     // 输出3
    
    Calculator::add(3, 4);
    cout << "累计和：" << Calculator::sum << endl;     // 输出10
    return 0;
}
```

这两种方法一般在统计类的对象的时候去使用.

# 8.纯虚函数和虚函数，重写

在一个类内，如果有函数定义类似于`virtual double getArea()=0;` ，即函数前面有virtual并且后面有=0，则这个函数就是一个纯虚函数，这个函数的定义强制要求在派生类里面去写；普通虚函数（前面有virtual，但是后面没有=0）的定义不用强制在派生类去写

如果一个类里面有至少一个纯虚函数（只要有纯虚函数,普通虚函数是不可以的），那么这个类被称为抽象类，==为了去理解抽象类：我认为把抽象类理解为“**类的半成品模板**”或“**规定了基本结构的模板**”是比较形象的。==

抽象类不能进行实例化
```C++
class Area {
public:
    virtual double getArea() = 0;  // 纯虚函数
};

int main() {
    // ❌ 错误：不能实例化抽象类
    // Area a;                    // 编译错误！
    // Area* p = new Area();      // 编译错误！
    
    // ✅ 正确：可以定义指针和引用
    Area* ptr;                    // 指针可以定义
    Area& ref = *ptr;             // 引用可以定义
    
    return 0;
}
```

派生类必须重写纯虚函数，如果抽象类里面有n个纯虚函数，那么派生类就必须重写n个纯虚函数，不然派生类也会变成抽象类

1. **纯虚函数**：`virtual double getArea() = 0;` 使 `Area` 成为抽象类
2. **不能实例化**：无法创建 `Area` 的对象，只能创建其派生类的对象
3. **必须实现**：派生类必须重写纯虚函数，否则派生类也会成为抽象类
4. **多态性**：可以通过基类指针调用派生类实现的 `getArea()` 函数  //!!!(没看懂，先不写)

抽象类中的纯虚函数在派生类中在定义也有要求,即有override标识符,用于在派生类中**显式声明**某个成员函数是要重写基类的虚函数。
```C++
class Base {
public:
    virtual void func1();
    virtual void func2(int x);
    virtual void func3() const;  // 普通虚函数
    virtual void func4() = 0;  // 纯虚函数也可以重写
};

class Derived : public Base {
public:
    // ✅ 正确：明确表示要重写 func1
    void func1() override {
        cout << "重写 func1" << endl;
    }
    
    // ✅ 正确：重写 func2
    void func2(int x) override {
        cout << "重写 func2, x = " << x << endl;
    }
    
    // ✅ 正确：重写 const 成员函数
    void func3() const override {
        cout << "重写 func3" << endl;
    }
    
    // ✅ 正确：重写纯虚函数
    void func4() override {
        cout << "实现纯虚函数" << endl;
    }
};
```
