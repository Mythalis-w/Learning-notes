4. char s\[80]; s="hello";这样赋值是可以的。 F

	- 在 C 语言中，数组名 `s` 是一个常量指针，代表数组的首地址，不能出现在赋值运算符的左边当作左值（不能重新指向另一个地址）。  
	- `"hello"` 是一个字符串字面量，它本身在内存中有自己的地址，因此 `s = "hello";` 试图把数组 `s` 的地址改成字符串字面量的地址，这是不允许的。
	- 如果一定要赋值的话，可以使用strcpy( s , "hello")来进行赋值

21. **选项A：NULL指针指向的内存地址是1**

	NULL指针的定义是**不指向任何有效内存地址**，其值通常为0（在大多数编译器中，`NULL`宏被定义为`(void*)0`），而非1。因此A错误。
	
	**选项B：可以安全地对NULL指针进行循环用操作**
	
	对NULL指针进行任何操作（如解引用`*p`、加减偏移等）都是**不安全的**，会导致程序崩溃（空指针引用异常）。循环操作也不例外，B错误。
	
	**选项C：NULL指针代表不指向任何有效内存**
	
	这是NULL指针的核心定义：指针变量的值为NULL时，明确表示它没有指向任何合法的内存区域（即“悬空”或“空”状态）。C正确。
	
	**选项D：定义指针时若未赋值,默认就是NULL指针**
	
	在C/C++中，**未初始化的指针变量会默认存储随机值**（即“野指针”），不会自动初始化为NULL。只有显式赋值（如`int* p = NULL;`）或通过动态分配（如`new`/`malloc`）后，指针才会有确定的状态。D错误
