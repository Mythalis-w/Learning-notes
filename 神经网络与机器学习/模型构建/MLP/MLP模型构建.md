这个是模型的核心，需要引入
模型构造我们使用 **PyTorch 框架**，继承 `nn.Module` 来搭建。

```python
class BasicMLP(nn.Module):  
    def __init__(self, input_dim: int) -> None:  
        super().__init__()  # 继承nn.Module的初始化方法  
        # 搭建MLP网络结构：输入层→隐藏层→输出层，加入激活函数和Dropout防过拟合  
        self.net = nn.Sequential(  
            nn.Linear(input_dim, 512),  # 输入层→第一隐藏层（输入维度input_dim，输出512维）  
            nn.ReLU(),                  # 激活函数，引入非线性，帮助模型学习复杂关系  
            nn.Dropout(0.25),           # Dropout层，随机丢弃25%的神经元，防止过拟合  
            nn.Linear(512, 256),        # 第一隐藏层→第二隐藏层（512维→256维）  
            nn.ReLU(),                  # 激活函数  
            nn.Dropout(0.2),            # 随机丢弃20%的神经元  
            nn.Linear(256, 128),        # 第二隐藏层→第三隐藏层（256维→128维）  
            nn.ReLU(),                  # 激活函数  
            nn.Linear(128, 1)           # 输出层（128维→1维，对应销量预测值）  
        )  
  
    # 前向传播：定义模型的计算流程，输入x，输出预测结果(就是正着走一遍)  
    def forward(self, x: torch.Tensor) -> torch.Tensor:  
        # squeeze(1)：删除多余的维度（将[batch_size, 1]转为[batch_size]），适配后续损失计算  
        return self.net(x).squeeze(1)
```

首先声明一个类，这个类继承父类`nn.Module`
然后写初始化函数，初始化函数里写要传入多少维（一般是有多少个特征就传入多少维，代码在后面的主程序自己去算；或者说，**输入维度 = 特征数量**，有多少列特征，就传入多少，代码会自动计算）
然后继承`nn.Module`的初始化方法

前面的写法可以算是固定写法了

比较关键的是后面数字的填写
这里注意
**特征越多、信息越复杂，前面和中间的神经元就要越大，慢慢收缩，最后才缩成 1，如果中间神经元过少，承载不下这么复杂的信息，就会出现严重的信息丢失，模型欠拟合，预测效果大幅下降。**

1. `nn.Linear(input_dim, output_dim)`
这个代码是一个线性的运算，相当于做了 $y=Wx+b$ 其中y是output，x是input，W是权重矩阵，b是偏置
****

2. `nn.ReLU()`
nn.ReLU()是把output里面的数字为负数的变成0,目的是**引入非线性**，让模型能学习复杂规律。
****

3. `nn.Dropout(num)`
这个代码是使得(num \* 100)% 的神经元失活，即全部为0

我们进行的2 and 3 看似都是在使得模型更加错误，但是实际上，我们可以在每一层把一些特征变成0，让模型去学习该层的某些数据，而不是学习全部，防止模型对某个数据依赖，同时还能引入非线性概念，多个线性变换依旧线性，引入非线性就能防止这个事情发生，然后通过优化器（梯度的运算等），去针对这参与运算的一些数据去调整权重矩阵和偏置，因为每一次Dropout的不一样，每一次都是非线性，不断地学习，去针对这一次的数据去做调整，从而得到一个比较好的W和b
****

4. `def forward(self, x):`
forward 是模型的**前向传播函数**
所有模型必须有 forward
`x` 就是输入数据（我们处理好的特征，这是一个类似于表格的东西）
****

5. `return self.net(x).squeeze(1)`
将x传入net网络里面去运算出结果，然后通过 `.squeeze(1)`去让这个维度变小
举例子
`[[1],[2],[3]] -> [1,2,3]`
