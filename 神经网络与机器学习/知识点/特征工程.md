# 1.日期特征
在对日期特征的提取中，对星期，月份等进行循环处理是很重要的（比如星期一和星期日，两个日期很近，但是模型会认为0和6是很远的）

日期处理一般处理：年、月、日、星期、天数、季度、对星期和月份的循环处理、是否周末、一年中的第几周（ISO日历）

具体的实现我们要先传入data的数据列，然后再做处理

```python
def add_date_features(df: pd.DataFrame) -> pd.DataFrame:
    out = df.copy()
    dt = pd.to_datetime(out["date"] , errors="coerce") # 将日期转换为datetime类型，这个是pandas用来提取日期的，
    out["year"] = dt.dt.year #提取年份，这里第一个dt是datatime的名字 , .dt是datatime的入口，.year则是提取年份 , `errors="coerce"` 表示异常日期会变成空值（`NaT`）
    out["month"] = dt.dt.month #提取月份，同上
    out["day"] = dt.dt.day #提取日期，同上
    out["dayofweek"] = dt.dt.dayofweek # 提取这一天是星期几，代码自动计算(0-6)
	    out["weekofyear"] = dt.dt.isocalendar().week.astype(int) # 提取一年中的第几周（ISO周），并转成整数。  .isocalendar()是转化为ISO日期，年周星期都换  .week是拿周  .astype(int)是转化为int
    out["quarter"] = dt.dt.quarter # 提取季度（1~4）
    out["dayofyear"] = dt.dt.dayofyear # 提取一年中的第几天（1~365/366）
    
    
    out["is_weekend"] = (out["dayofweek"] >= 5).astype(np.int64) # 主要解释np.int64，这个是numpy的形式，转化为bool值。 记得放在numpy库中，放了删掉这句话
    out["month_sin"] = np.sin(2 * np.pi * out["month"] / 12.0) # 下面的代码纯粹就是ai学习用的一些数学周期方面的东西，这里在下面也讲一下
    out["month_cos"] = np.cos(2 * np.pi * out["month"] / 12.0)
    out["dow_sin"] = np.sin(2 * np.pi * out["dayofweek"] / 7.0)
    out["dow_cos"] = np.cos(2 * np.pi * out["dayofweek"] / 7.0)
    out = out.drop(columns=["date", "id"], errors="ignore") # 删除data和id这两列，如果报错就忽略
    return out
```

处理日期的方式来自pandas库
这里我们讲空行下面的部分，上面是pandas的内容

1. `out["is_weekend"] = (out["dayofweek"] >= 5).astype(np.int64)`

这个是在out中搞一个新列叫做`is_weekend`，主要针对于是否是周末进行处理，`(out["dayofweek"] >= 5)` 会判断星期是否大于等于5（0-6），然后返回bool值，然后后面的代码是把 bool值转为int值，即 0 or 1
****
2. 
```python
    out["month_sin"] = np.sin(2 * np.pi * out["month"] / 12.0) 
    out["month_cos"] = np.cos(2 * np.pi * out["month"] / 12.0)
    out["dow_sin"] = np.sin(2 * np.pi * out["dayofweek"] / 7.0)
    out["dow_cos"] = np.cos(2 * np.pi * out["dayofweek"] / 7.0)
```
这个代码是 周期性编码：用sin/cos处理月份、星期，体现环形周期性
至于一个特征要用sin和cos两个三角函数的原因

`sin(θ) = sin(π-θ)`，会出现不同位置同一个值，也就是说只用 `sin` 时，模型可能分不清两个不同月份位置，但是`sin` 和 `cos` 组成二维坐标 `(sinθ, cosθ)`，使得每个特征都有一个独一的点
****
3. `out = out.drop(columns=["date", "id"], errors="ignore")`

这个是把data和id删除，因为data已经被我们做完处理了，所以删除，id是无关序号，删除不要


# 特征处理
在提取完特征后，我们也需要对特征进行一定的预处理，从而整合所有特征，为模型输入做准备

```python
def build_features(  
    train_df: pd.DataFrame, test_df: pd.DataFrame  
) -> tuple[np.ndarray, np.ndarray, np.ndarray, pd.Series]:  

    # 提取训练集的目标值（销量num_sold），转为float32格式（适配PyTorch）  
    y = train_df["num_sold"].to_numpy(dtype=np.float32)  
    # 提取训练集的日期（用于后续划分验证集）  
    train_dates = pd.to_datetime(train_df["date"], errors="coerce")  
  
    # 对训练集、测试集分别提取日期特征（训练集需先删除目标列num_sold，避免数据泄露，因为y已经存储了num_sold了）  
    x_train_df = add_date_features(train_df.drop(columns=["num_sold"]).copy())  
    x_test_df = add_date_features(test_df.copy())  
  
    # 合并训练集和测试集特征（统一做独热编码，避免训练集/测试集编码不一致）  
    full = pd.concat([x_train_df, x_test_df], axis=0, ignore_index=True)  
    # 对分类特征（country、store、product）做独热编码，转为数值型（MLP仅接受数值输入）  
    full = pd.get_dummies(  
        full,  
        columns=["country", "store", "product"],  # 需要编码的分类列  
        dummy_na=False,                          # 不处理NaN（后续已填充）  
    ).fillna(0.0)                                # 填充缺失值为0.0  
  
    # 转为float32格式的数组，适配PyTorch  
    x_all = full.astype(np.float32).values  
    # 拆分回训练集、测试集特征（按原始训练集长度拆分）  
    x_train = x_all[: len(x_train_df)]  
    x_test = x_all[len(x_train_df) :]  
  
    # 标准化：用训练集的均值和标准差处理，避免测试集数据泄露（仅用训练集统计量）  
    mean = x_train.mean(axis=0, keepdims=True)  # 训练集各特征的均值  
    std = x_train.std(axis=0, keepdims=True)    # 训练集各特征的标准差  
    std[std < 1e-6] = 1.0                      # 避免标准差为0导致除零错误  
    x_train = (x_train - mean) / std            # 训练集标准化  
    x_test = (x_test - mean) / std              # 测试集标准化（用训练集的均值/标准差）  
  
    # 返回处理好的训练特征、目标值、测试特征、训练集日期  
    return x_train, y, x_test, train_dates
```

这个特征处理，主要是起到防止数据泄露，划分训练集和测试集等作用
我们传入train.csv和test.csv做处理
提取num_sold存储到y，做单独的数据存储，防止数据泄露，并在后面参与损失的计算
****
`train_dates = pd.to_datetime(train_df["date"], errors="coerce")`
这个代码是提取训练集的日期，为后续划分训练集等做铺垫

one-hot：主要是用作处理字符串的形式，因为MLP模型不能理解字符串，我们要把字符串转为数字去进行处理，这就是one-hot所做的事情，举个例子，原本 country 这一列有3个国家，分别是A、B、C这三个国家，在one-hot中，country这一列将被A、B、C所代替，此时如果表示A国家，A下面的数字就是1，B、C下面的数字就是0。
下面这个一串代码就是one-hot的过程，准确来说是 pd.get_dummies（）
